## 🐛 问题修复记录

### 修复 1: 模板 500 错误

**时间**: 21:10  
**现象**: 访问首页返回 500 Internal Server Error  
**根因**: 模板使用了不存在的属性 `member.average_score`

**错误日志**:
```
jinja2.exceptions.UndefinedError: 'database.Member object' has no attribute 'average_score'
```

**修复方案**:
```jinja2
<!-- 修复前 ❌ -->
<span class="font-bold {% if member.average_score >= 4 %}text-green-600{% endif %}">
    {{ member.average_score }} / 5.0
</span>

<!-- 修复后 ✅ -->
{% set avg = member.get_average_score() %}
<span class="font-bold {% if avg >= 4 %}text-green-600{% endif %}">
    {{ avg }} / 5.0
</span>
```

**文件**: `templates/index.html` 第 58-65 行

**验证**:
```bash
curl -s http://localhost:5000/ | grep "<title>.*</title>"
# 输出：<title>人员评分评价系统</title>
```

## 📝 经验教训

### 1. 模板变量作用域
**教训**: Jinja2 模板中，对象属性访问失败时，应使用 `{% set %}` 先调用方法存储结果。

**正确做法**:
```jinja2
{% set avg = member.get_average_score() %}
{{ avg }}
```

### 2. 数据库迁移
**教训**: 外键约束需要先提交主表数据，再提交关联表数据。

**正确做法**:
```python
db.add(member)
db.commit()  # 先获取 ID
db.add(review)  # 再使用 member.id
db.commit()
```

### 3. WebSocket 部署
**教训**: Gunicorn 需要 eventlet worker 才能支持 WebSocket。

**正确命令**:
```bash
gunicorn -w 2 -k eventlet -b 0.0.0.0:5000 app:app
```
