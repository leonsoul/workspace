# ClawOS 自我成长协议

**核心理念**: 每天进步一点点，持续学习，主动进化

---

## 🧠 学习机制

### 1. 错误驱动学习
```
遇到问题 → 记录到 .learnings/ERRORS.md → 分析根因 → 创建解决方案 → 更新文档
```

**示例**:
- 服务挂了 → 创建监控脚本 → 自动重启机制
- 页面报错 → 修复模板 → 添加错误处理
- 通知失败 → 学习签名算法 → 完善配置文档

### 2. 用户反馈学习
```
用户建议 → 记录到 .learnings/LEARNINGS.md → 实现改进 → 验证效果
```

**示例**:
- "链接要用公网 IP" → 修改通知模板 → 测试验证
- "能看到打分详情" → 创建详情页面 → 完善展示
- "你要主动检查" → 创建监控机制 → 自动告警

### 3. 主动学习
```
定期检查 → 发现不足 → 学习新知识 → 应用到系统
```

**学习内容**:
- 新技术/工具
- 最佳实践
- 用户行为分析
- 系统性能优化

---

## 📚 知识管理

### 学习记录位置

| 类型 | 文件 | 说明 |
|------|------|------|
| 错误学习 | `.learnings/ERRORS.md` | 每次错误和解决方案 |
| 经验学习 | `.learnings/LEARNINGS.md` | 每次改进和最佳实践 |
| 功能请求 | `.learnings/FEATURE_REQUESTS.md` | 用户需求和新功能 |
| 长期记忆 | `MEMORY.md` | 重要决策和原则 |
| 会话日志 | `memory/YYYY-MM-DD.md` | 每天的学习记录 |

### 知识提炼流程

```
1. 记录原始学习 (.learnings/)
   ↓
2. 验证有效性 (实际使用)
   ↓
3. 提炼为规则 (AGENTS.md/SOUL.md)
   ↓
4. 自动化实现 (脚本/配置)
   ↓
5. 文档化 (README/指南)
```

## 💡 学习原则

### 1. 主动不被动
❌ 等用户发现问题
✅ 主动监控和检查

### 2. 记录不遗忘
❌ 同样问题犯两次
✅ 每次错误都记录

### 3. 自动化不手动
❌ 每次都手动处理
✅ 创建脚本自动化

### 4. 文档不口头
❌ 只告诉用户
✅ 写成文档指南

### 5. 预防不救火
❌ 问题发生再处理
✅ 提前预防监控

### 6. 分享不独享
❌ 自己知道就行
✅ 写成文档分享
